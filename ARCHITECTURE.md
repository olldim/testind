# Архітектура додатку

## Загальна структура

Додаток складається з двох частин:
1. **Frontend**: Flutter мобільний додаток
2. **Backend**: Node.js Express API сервер

```
┌─────────────────────────────────────────────────────────────┐
│                     Flutter App                              │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  UI Layer (Screens & Widgets)                        │   │
│  │  - LoginScreen                                       │   │
│  │  - HomeScreen (з 3 табами)                          │   │
│  └──────────────────────────────────────────────────────┘   │
│                           ↓                                  │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Business Logic (Providers)                          │   │
│  │  - AuthProvider (управління авторизацією)          │   │
│  └──────────────────────────────────────────────────────┘   │
│                           ↓                                  │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Services & Storage                                  │   │
│  │  - ApiService (HTTP запити)                         │   │
│  │  - StorageService (локальне сховище)               │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                           ↓ HTTP/HTTPS
┌─────────────────────────────────────────────────────────────┐
│                    Node.js Backend                           │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  API Routes                                          │   │
│  │  - /api/auth/login                                 │   │
│  │  - /api/news                                        │   │
│  │  - /api/polls                                       │   │
│  └──────────────────────────────────────────────────────┘   │
│                           ↓                                  │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Controllers (Business Logic)                        │   │
│  │  - AuthController                                   │   │
│  │  - NewsController                                   │   │
│  │  - PollsController                                  │   │
│  └──────────────────────────────────────────────────────┘   │
│                           ↓                                  │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Middleware                                          │   │
│  │  - Authentication (JWT verification)               │   │
│  └──────────────────────────────────────────────────────┘   │
│                           ↓                                  │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  Database (MySQL)                                    │   │
│  │  - users                                            │   │

└─────────────────────────────────────────────────────────────┘
```

```
           ↓

           ↓

           │
           ↓
```

## Потік отримання новин

```
┌─────────────────────┐
│ Користувач відкриває │
│ вкладку "Новини"    │
└──────────┬──────────┘
           │
           ↓
┌─────────────────────────────────┐
│ NewsTab запускає FutureBuilder  │
│ та викликає ApiService.getNews()│
└──────────┬──────────────────────┘
           │
           ↓
┌─────────────────────────────────────────┐
│ GET /api/news                            │
│ Header: Authorization: Bearer {token}   │
└──────────┬──────────────────────────────┘
           │
           ↓
┌─────────────────────────────────────┐
│ Middleware.authenticateToken()      │
│ перевіряє JWT токен                 │
└──────────┬──────────────────────────┘
           │ ✅ Валідний
           ↓
┌────────────────────────────────────┐
│ NewsController.getNews() отримує:  │
│ 1. Список новин                    │
│ 2. Картинки для кожної новини      │
│ 3. Посилання для кожної новини     │
└──────────┬───────────────────────┘
           │
           ↓
┌─────────────────────────────────────┐
│ Повертає JSON з новинами            │
└──────────┬──────────────────────────┘
           │
           ↓
┌─────────────────────────────────────┐
│ NewsTab отримує дані та показує     │
│ список новин в UI                   │
└─────────────────────────────────────┘
```

## Потік голосування

```
┌─────────────────────────────┐

│ варіант відповіді в опитуванні│
└──────────┬──────────────────┘
           │
           ↓

           │
           ↓

└──────────┬───────────────────────────────────────┘
           │
           ↓
┌──────────────────────────────────────────────────┐
│ Повертає успіх (200 OK)                          │
└──────────┬───────────────────────────────────────┘
           │
           ↓
┌──────────────────────────────────────────────────┐
│ PollsTab перезавантажує список голосувань       │
│ та оновлює UI з новими даними                   │
└──────────────────────────────────────────────────┘
```

## Security (Безпека)

### Аутентифікація
- **Метод**: JWT (JSON Web Tokens)
- **Тривалість**: 7 днів
- **Збереження**: SharedPreferences (зашифровано на рівні ОС)

### Паролі
- **Хешування**: bcryptjs (10 раундів)
- **Не зберігаються**: Паролі ніколи не зберігаються в plaintext

### HTTPS
- В продакшені використовуйте HTTPS замість HTTP
- На розробку можна використовувати HTTP

### Валідація
- Всі вхідні дані валідуються на сервері
- JWT токени перевіряються перед кожним запитом

## Масштабування та оптимізація

### Поточні обмеження
- Всі картинки завантажуються через URL
- Нема кешування на клієнті
- Нема пагінації для новин

### Можливі покращення
1. **Кешування**: Додати кешування картинок та даних
2. **Пагінація**: Завантажувати новини порціями
3. **WebSocket**: Для real-time оновлень новин
4. **Cloud Storage**: Для завантаження картинок (AWS S3, etc.)
5. **CDN**: Для швидшої доставки картинок
6. **Database Indexes**: Добавити індекси для швидших запитів

## Розробка та debugging

### Корисні команди

```bash
# Backend debugging
NODE_DEBUG=* npm start

# Flutter debugging
flutter run -v

# MySQL queries
mysql -u root -p
SELECT * FROM news;
SELECT COUNT(*) as total_votes FROM poll_votes WHERE poll_id = 1;
```

### Tools для тестування API
- **Postman**: https://www.postman.com/
- **Thunder Client**: VS Code extension
- **curl**: Command line

### Приклад тестування API з curl

```bash
# Вхід
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"login":"student1","password":"password"}'

# Отримання новин
curl -X GET http://localhost:3000/api/news \
  -H "Authorization: Bearer YOUR_TOKEN"

# Голосування
curl -X POST http://localhost:3000/api/polls/1/vote \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"option_id":1}'
```
